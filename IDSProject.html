AI-Based Intrusion Detection System (IDS)

Network anomaly detection using classical ML on UNSW-NB15, tuned Random Forest, and a homelab deployment path with Suricata/Snort.

Python scikit-learn RandomizedSearchCV UNSW-NB15 Ubuntu VM Suricata/Snort (planned)

Overview

I built an IDS that flags anomalous network activity using a supervised ML pipeline. It ingests tabular flow features, performs preprocessing, and predicts normal vs. malicious traffic. The goal is a practical, SOC-relevant project demonstrating data wrangling, modeling, and deployment.

Why It Matters

  • Hands-on ML for security: feature engineering, training, evaluation.
  • Operational focus: designed to plug into Suricata/Snort for real alerts.
  • Homelab ready: runs on an Ubuntu VM; lightweight and reproducible.

Data & Features

  • Dataset: UNSW-NB15 (mix of normal & multiple attack types).
  • Preprocessing: missing-value handling, label encoding, train/test split, scaling as needed.
  • Targets: Binary (malicious vs normal). Multi-class can be added later.

Model & Training

  • Algorithm: Random Forest (fast, robust, interpretable feature importance).
  • Tuning: RandomizedSearchCV over depth, estimators, splits, bootstrap.
  • Validation: Stratified K-fold cross-validation.
  • Artifacts: Persisted model with joblib.

Deployment & Use

  1. Clone repo & create venv; install requirements.
  2. Run training script or load prebuilt .joblib model.
  3. Feed captured flow features (CSV or live pipe) to the predictor.
  4. Emit alerts to file/stdout; integrate with Suricata/Snort or a SIEM.
# sample run (training)
python train.py --data data/UNSW-NB15.csv --out model.joblib

# sample run (inference)
python predict.py --model model.joblib --in flows.csv --out alerts.json
